🔴 2️⃣ project_hash Strategy Is Weak

You said:

SHA-256 of sorted file list + mtimes

This is fragile.

If file content changes but mtime doesn’t update → no detection.

L9 improvement:

Hash file contents for smaller files

Use content fingerprint sampling for large files

Store project_hash + file_count

This matters for re-index correctness.

🔴 3️⃣ Hallucination Guard Is Too Naïve

Your current guard:

Regex: [A-Z][A-Z0-9_]{2,}


Problems:

False positives

Misses lowercase tags

Misses scoped tags (Program:Tag)

Misses dotted tags (Motor.Speed)

Better approach:

Extract candidates

Compare against actual tag set using fuzzy matching threshold

Only fail if candidate not in tag index

This needs improvement.
